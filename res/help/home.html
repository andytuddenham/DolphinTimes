Dolphin Times Help

The Dolphin Times program runs alongside the Dolphin Timer, reading the files produced by the Dolphin Timer and sorting them in 'finish time' rather than 'lane number' order. This makes it easier for the referees to check that their finish order matches the Dolphin Timer.

The main window

The main window contains a number of race results, each race result has the race number at the top left, a table below that has details of the times swum by each lane arranged in 'finish time' order. There is also a 'View...' button at the top right which opens a new window containing just that race result.

The number of race results in the main window can be set in the DolphinTimes.properties file (see below). When this number is reached, new results are added at the top left and old ones are dropped.

Settings

The program is controlled by a number of settings in the DolphinTimes.properties file, this file is read when the program starts.

The file contains a number of setting in 'key=value' format. Each 'key=value' pair must be on a separate line with no spaces.

The following 'key=value' pairs can be used:
keydefaultdescription
lane.count10specifies the number of lanes in the pool
race.countallspecifies the number of results to keep in the main window (must be an integer)
header.font.size11specifies the size of the text displaying the race number in the results header, must be between 8 and 24
table.font.size11specifies the size of the text displaying the race results in the table, must be between 8 and 24
font.size11specifies the size of the text if header.font.size or table.font.size are not specified, must be between 8 and 24

any line starting with a '#' is treated as a comment and ignored by the program.

Here is an example:

lane.count=8
race.count=6
header.font.size=16
table.font.size=24
#font.size=11

If any of these settings are changed, the program will need to be restarted before they will become effective.
